# Vercel Deployment Guide

Last updated: 2026-06-11

## 1. Production Build

The project has been checked with:

```bash
npm run build
```

Expected result:

- Build completes successfully.
- TypeScript validation passes during the build.
- Static pages and API routes are generated by Next.js.

## 2. Environment Variables

Create these variables in Vercel:

Project Settings > Environment Variables

```bash
FOOTBALL_DATA_API_KEY=
FOOTBALL_DATA_COMPETITION=WC

THE_ODDS_API_KEY=
THE_ODDS_API_SPORT=soccer_fifa_world_cup
THE_ODDS_API_REGIONS=eu
THE_ODDS_API_MARKETS=h2h,totals,spreads
THE_ODDS_API_BOOKMAKER=

AI_PROVIDER=deepseek
AI_API_KEY=
AI_BASE_URL=https://api.deepseek.com
AI_MODEL=deepseek-v4-flash

APP_PASSWORD=
AUTH_SECRET=
```

Optional legacy fallback variables:

```bash
DEEPSEEK_API_KEY=
DEEPSEEK_MODEL=deepseek-v4-flash
OPENAI_API_KEY=
OPENAI_MODEL=gpt-5-mini
```

Do not upload `.env.local`. It is only for local development.

`APP_PASSWORD` is optional but recommended for private public deployments. If it is empty, the site is open. If it has a value, every page and API route requires the password first.

`AUTH_SECRET` is optional. Set it to a long random string to make the access cookie harder to forge.

Recommended settings:

- Framework Preset: Next.js
- Install Command: `npm ci`
- Build Command: `npm run build`
- Output Directory: leave empty
- Node.js Version: Vercel default or latest supported LTS

The project includes `vercel.json` with:

- Next.js framework setting
- Production build command
- `npm ci` install command
- 30-second limits for external API routes

## 4. Password Protection

Because this app calls paid or quota-limited APIs, use password protection if the deployment is only for yourself.

Recommended app-level option:

- Set `APP_PASSWORD` in Vercel Environment Variables.
- Set `AUTH_SECRET` to a long random string.
- Redeploy the project.
- Visitors must enter the password before they can use pages or API routes.

Recommended Vercel platform options:

- Vercel Deployment Protection for preview and production deployments.
- Vercel team/project password protection if your plan supports it.
- Keep the production URL private if password protection is not available.

Basic setup path in Vercel:

1. Open the Vercel project.
2. Go to Settings.
3. Open Deployment Protection or Security.
4. Enable protection for Production and Preview deployments.
5. Add the password or allowed login method.
6. Redeploy if Vercel asks for it.

## 5. Secret Safety Check

Before deploying:

- `.env.local` must stay local.
- `.env.example` must contain placeholders only.
- API keys must be set only in Vercel environment variables.
- Do not use `NEXT_PUBLIC_` for private API keys.

Current secret scan result:

- No hardcoded real API keys were found outside ignored local env files.

## 6. Post-Deployment Smoke Test

After Vercel deploys:

1. Open `/settings`.
2. Confirm Football Data API shows configured.
3. Confirm The Odds API shows configured.
4. Confirm AI API shows configured.
5. Sync matches.
6. Sync odds.
7. Open a match detail page.
8. Generate one AI analysis.
9. Add one bankroll record.
10. Export a backup.

## 7. Important Production Notes

This app is ready for private personal deployment.

Before sharing it publicly, add:

- Login or deployment protection.
- API rate limiting.
- Persistent database storage if data must work across browsers or devices.
- Monitoring for API usage and AI cost.
